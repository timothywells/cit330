Reading

This week the reading covered Further Functions and AJAX. Honestly the reading was interesting. First with the Further Functions, it showed me that there are way to take what we have learned and condense that code down so that instead of having repetitive code you can make it conditional. Ironically I used some of those further functions like setting constants that get items by ID and Class in my Todo App that I have been trying to finish. Another thing that I added that was similar is event listeners. Something that I have already used that is related to further functions is the try/catch but in my Back End Class so I am a little familiar with how it works. All in all the Further Functions chapter was interesting and hopefully will make the next project a little easier than the first app.

Now as for AJAX, it’s a little difficult to understand but that is because it is just a little different than traditional JS. AJAX is used to communicate with a server much like NODE does. Which both do the same things in the aspect of communication with a server. As I read through the chapter I came to find out more about how AJAX helps traditional JavaScript to communicate with the server. From the video that I watched on it, not related to the book, it explained that JS is a front end language, SQL and other serverlanguages are the back end while AJAX and NODE are more a middle man that allows the front end and back end to communicate with each other. One of the many things they do is grab information from an API which stands for Application Programming Interface. An API, to my understanding, is a database of information that can be accessed using AJAX and NODE from a website. What I know is this, I will be using AJAX a bit more in my other class and but will likely use the same project for both classes with the difference being one will be deployed on the front end, this class, the other will be deployed on my server, the back end.
