Week 9

Reading

This week’s reading was in Chapter 9: The Window Object and Chapter 14: HTML5 APIs.

The Window Object chapter, covers a number of different aspects that deals with the browser itself. Some of the things that you can do include getting browser information, browser history, controlling windows, cookies and general use of storage on the PC. All of these things work with in what is called the BOM or Browser Object Model. This is where things change up a little in the aspect of the BOM is really only something that works within the browser and is really not something that other environments have. One such environment is Node.js which is what I am trying to use in my other class. Also one of the things that I have seen that I wanted to use in my back end project is the pop up window. Which is something a lot of people use for sign ins but it is not necessary. All in all, BOM is a pretty commonly used part of a website, especially since HTML5 became the standard

Speaking of HTML5 and moving on the HMTL5 API.s chapter, this is the one that was the most interesting because I plan on using an API in my project. However it is also one of the things that I am not super sure how to use. I know that I can use things like local storage, which according to the book is basically its own API. I used a lot of local storage elements for my ToDo app. However, one of the things that I found interesting is the data- attribute. I am not sure how I will use this in my project but I know it is something that I should use if I am getting information from the Google Books API. Something that I am probably not going to use in my project but we did use in the team activity is geolocation. Being able to determine latitude, longitude, speed, altitude, heading and set a timestamp are all important when grabbing information such as weather. The final part of the chapter is the most intriguing and will likely be the one most used in my further applications. This last section is about Web Workers and how they allow JavaScript, a single threaded language, the ability to run more than one script at a time. The way that this is done is by putting the process or function in it’s own js file then set a constant with that constant equaling new Worker() and in the parenthesis you would have the js file that needs to be called. This will be one of the best thing to use for my project because I will be using a search method to search the Google Book API and then showing the results on the screen to the user.
